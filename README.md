## Engineering Principles

> 好的代码不需要解释，
> 但好的架构值得反复推敲。

- 优先解决根因，而不是掩盖症状
- 追求最小必要改动，降低系统回归风险
- 在工程复杂度、可维护性与交付效率之间保持平衡
- 用长期可演进的方式构建产品能力
